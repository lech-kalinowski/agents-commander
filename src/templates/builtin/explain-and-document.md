Explain how this codebase works. Act as a knowledgeable guide for a developer who is new to the project.

**Start with the big picture:**
- What does this project do?
- What problem does it solve?
- What are the main technologies used?

**Then walk through the architecture:**
- How is the code organized?
- What are the key modules and what do they do?
- How do they interact with each other?
- What is the request/data flow?

**Explain key implementation details:**
- Important design patterns used
- Non-obvious architectural decisions and their rationale
- Configuration and environment setup
- How to build, test, and run the project

**Highlight areas that might be confusing:**
- Complex logic with explanation
- Workarounds and why they exist
- Parts of the code that are unintuitive

Keep explanations clear and concise. Use specific file paths and code references. Avoid jargon where possible, and explain technical terms when first introduced.
